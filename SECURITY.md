# Security Policy

Mapsmith is an early local-first editor. The core app is designed to run without accounts or cloud storage, but security reports are still welcome.

## Supported Versions

Until Mapsmith has tagged releases, security fixes target the `main` branch.

## Reporting a Vulnerability

Please avoid posting sensitive exploit details, private diagrams, credentials, tokens, or personal data in public issues.

If GitHub private vulnerability reporting is enabled for this repository, use that channel. If it is not available, open a minimal public issue that says you have a security concern without including exploit details or sensitive data.

Useful safe details include:

- affected commit or branch,
- browser and operating system,
- high-level impact,
- whether the issue affects `.mapsmith`, SVG, or PNG handling,
- and whether a sanitized reproduction can be shared.

## Public-Safe Fixtures

Security and export/import reports should use small demo files. Do not include real business diagrams, private URLs, local absolute paths, API keys, tokens, cookies, or passwords.
